ABSTRACT

PhishLens is a secure full-stack platform that combines AI-assisted phishing email detection, explainable risk outputs, and role-based workflows for users, researchers, and administrators. The system uses React/Vite, Express, MongoDB, and FastAPI ML services. BERT, DistilBERT, and LLaMA models were trained on 18,000 emails and compared with five ensemble methods. The best method, Stacking Random Forest, achieved 98.44% accuracy and an F1 score of 0.9805.

INTRODUCTION
  • Phishing emails exploit trust, urgency, fear, reward, and familiar brands.
  • Rule-based filters can miss changing wording and new attack patterns.
  • Secure design is needed because predictions, dashboards, and logs contain sensitive information.
PhishLens combines model evaluation with explainable detection, RBAC, 2FA, dataset workflows, and testing evidence.
OBJECTIVES
  • Classify email content and explain suspicious indicators.
  • Train and compare BERT, DistilBERT, and LLaMA on 18,000 emails.
  • Combine models using five ensemble methods.
  • Implement bcrypt, JWT, RBAC, and optional TOTP 2FA security.
  • Support researcher and admin dashboards with activity logging.
DATASET AND METHODOLOGY
  • 18,650 Kaggle emails were prepared, with 18,000 used for testing and model work.
  • TF-IDF vectorisation and domain-specific features were engineered.
  • Dataset split: 70% train, 15% validation, and 15% test.

Design principle: the browser never calls the ML service directly. Express validates requests, enforces RBAC, stores audit records, and returns JSON-only responses.
SECURITY CONTROLS
  • bcrypt password hashing and JWT login
  • TOTP-based optional 2FA setup
  • Role middleware for admin, researcher, and user
  • CSV-only upload and request validation
  • MongoDB operator sanitisation
ROLE DASHBOARDS
  • User: submit emails and view predictions.
  • Researcher: upload CSV datasets and compare model metrics.
  • Admin: approve users, assign roles, and view platform logs.

CONCLUSION AND FUTURE WORK

PhishLens demonstrates a complete academic secure AI prototype: model comparison, ensemble learning, explainable prediction, secure role-based workflows, and repeatable testing evidence.

Future work includes production BERT and DistilBERT weights, SPF and DKIM intelligence, Docker Compose, CI/CD scanning, refresh tokens, password reset, and monitoring.
